本地 LLM 评测实录：7B 模型真的比 0.5B 聪明吗？

用 llama.cpp 在本地 CPU 对比测试多个开源模型，发现一个反直觉的结论
April 14, 2026 · 2 min · 229 words

花了 3 天时间测试本地大模型，结论是：7B 只比 0.5B 聪明约 10%，但代价是 15 倍体积和 20 倍加载时间。

背景#

学完 FastAPI 和 Graph RAG 后，我陷入了"工具焦虑"——知道很多概念，但不知道本地跑模型到底是什么体验。于是决定：亲手测一测。

测试环境#

  • CPU: Intel i9（16 线程）
  • 内存: 32GB
  • 框架: llama-cpp-python
  • 模型格式: GGUF（4-bit 量化）

参测模型#

模型参数量文件大小特点
Qwen2.5-0.5B-Instruct0.5B~1GB极轻量，速度快
Qwen2.5-1.5B-Instruct1.5B~3GB中文优化
Phi-22.7B~4GB微软出品，英文强
TinyLlama-1.1B1.1B~638MB小巧精悍
DeepSeek-R1-7B7B~15GB带思维链，推理强
Qwen2.5-7B-Instruct7B~15GB中文大模型

评测方法#

没有用什么高大上的 Benchmark，而是设计了 20 道贴近实际的题目，覆盖：

  • 逻辑与推理：序列推理、逻辑陷阱、悖论识别
  • 语言理解：歧义消解、隐喻理解、反讽检测
  • 数学思维：抽象代数、条件概率、无限序列
  • 常识推理：物理直觉、生物节律、社会常识
  • 创造性思维：概念融合（如"区块链 + 光合作用"）
  • 伦理与价值观：自动驾驶伦理困境
  • 元认知：自我反思能力

评分采用双机制：

  1. 规则评分：检查答案是否包含关键词
  2. LLM 评判：用更强的模型（Kimi）作为裁判打分

关键发现#

1. 性价比暴死

为了 10% 的提升（32% → 43%），需要：

  • 多等 20 分钟下载
  • 多占用 14GB 磁盘/显存
  • 推理速度从 62 tok/s 降到 10 tok/s

2. 全军覆没

所有本地模型在数学、逻辑、文化题上集体不及格。它们更擅长"胡说但看起来有道理"。

3. 话痨税

DeepSeek-R1 的思维链写了 3929 token，只比 0.5B 高 7 分。对于简单任务，思维链的额外开销得不偿失。

模型选择建议#

根据测试结果，我的建议：

场景推荐模型
快速原型验证Qwen0.5B（速度最快）
中文日常问答Qwen1.5B（性价比最佳）
英文任务Phi-2（微软训练质量高）
资源极度受限TinyLlama（638MB 全能跑）
复杂推理直接调 API（Kimi/DeepSeek/GPT-4）

给后来者的建议#

  1. 别折腾本地 7B 了。简单任务用 0.5B~1.5B，复杂推理直接调 API。本地部署的最大价值是离线可用和数据隐私，不是性能。

  2. 测试的价值不是看模型多聪明，是知道它什么时候会胡说。我整理了一份"禁区清单"：

    • 数学证明题（必错）
    • 需要中国文化背景的隐喻（大概率错）
    • 反常识陷阱题（容易踩）

  3. 动手比等待重要。等 15GB 模型下载的焦虑，远大于写 Python 分析的快乐。边下载边写代码，时间不会浪费。

代码与工具#

评测框架已开源，支持：

# 快速测试本地模型
python cli.py --model Qwen/Qwen2.5-0.5B-Instruct

# 测试 API 模型
python cli.py \
    --api-key $KIMI_API_KEY \
    --base-url https://api.moonshot.cn/v1 \
    --model moonshot-v1-8k \
    --report md --output report.md

完整代码：ai-learning/Day05_My_Gpu_Icrying

这次评测让我对本地 LLM 彻底祛魅。它们不是万能的，但在合适的场景下（边缘设备、离线环境、简单任务），小模型依然有大价值。
